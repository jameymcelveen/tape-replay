Updates & install

TapeReplay ships as a bundled Electron app with a self-contained .NET backend. UI-only fixes can arrive via CDN patches between full installer releases.

Install from release

  • macOS — open the DMG, drag TapeReplay to Applications.
  • Windows — run the NSIS setup exe from the release page.

Build locally: make installer-mac or make installer-win (on Windows).

CDN layout (tapereplay.surge.sh)

URL pathContents
/manifest.jsonJS patch auto-update manifest
/patch_ver_X.Y.Z.zipFrontend-only patch bundle
/help/This documentation site

JS patch updates

On startup the app checks manifest.json. If a newer patch version exists, it downloads and unpacks the zip into user data — no full reinstall required.

make cdn-dist PATCH=0.1.1 SURGE_DOMAIN=tapereplay.surge.sh
cd dist && surge . tapereplay.surge.sh

Patch deploys include the built frontend zip and the help/ folder.

Version display

The app header shows installer version and patch version when running the Electron shell (e.g. v0.1.0 patch 0.1.1).

Major/minor bumps need a new installer build. Patch bumps are CDN-only. CI builds installers only on major/minor tags.

Where data lives (installed app)

SQLite and downloaded patches live under the OS user-data folder for TapeReplay, not inside the application bundle. Your recorded bars persist across app updates.
