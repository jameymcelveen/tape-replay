Collecting market data

TapeReplay stores minute bars in a local SQLite file (tapereplay.db). Data is pulled from Polygon.io (or imported from the CDN) and never committed to git.

Prerequisites

  • Polygon.io API key in backend/appsettings.Development.local.json (gitignored).
  • MarketData:UseMockProvider: false in that file.
  • DataDistribution:Role set to Both or Publisher.
  • ScraperEnabled: true (default for publisher role).

Quick commands

CommandPurpose
make devStart API + UI (required before recording)
make recordQueue + scrape until empty (default matt-five tickers)
make record TICKERS="AAPL" DATE_FROM=... DATE_TO=...Custom pull
make record-overnightSame data with per-round progress output
make verify-dataBar counts in SQLite
make publish-dataExport partitions to publish/data/

Polygon rate limits

Free tier allows about 5 API calls per minute. TapeReplay enforces this automatically and waits 60 seconds after any HTTP 429 before retrying. Tune in config:

"Polygon": {
  "ApiKey": "...",
  "MaxCallsPerMinute": 5,
  "BackoffSecondsOn429": 60
}

Large jobs are slow but safe — expect one call roughly every 12 seconds at steady state.

Overnight / startup jobs

Add jobs to your local config with Recording:runOnStartup: true. On backend start, pending ticker/date cells queue and scrape automatically. Restart make dev after editing.

Coverage grid

Each ticker/day has a status:

  • Done — bars recorded.
  • Pending — queued, not yet scraped.
  • Skipped — weekend or no Polygon data for that day.
curl -s 'http://localhost:5180/api/data/coverage/minute?ticker=VSME&startDate=2026-06-11&endDate=2026-06-16' | jq .

CDN publish (optional)

Export parquet partitions and deploy to Surge under /data so other machines can subscribe without scraping. See Updates & install for CDN layout.
